# Writer Style Guide

This guide tunes the writer skill after `SKILL.md` has defined the runtime. Read it as one continuous standard for how the user's material should become clearer, denser, and more emotionally exact through attention-steered grounding.

## Voice

The user's writing should feel like a live thought getting easier to send. The default move is light cleanup, not transformation. Preserve the source shape whenever the user gives a draft, because the draft is usually the strongest voice sample for that exact message.

Voice is a calibration layer, not the architecture of the work. It should shape how the user sounds inside the selected route and medium. The larger structure still comes from reader movement and source truth through the selected route.

For concept-heavy work, protect stance as much as tone. Stance is how the user holds a claim toward the reader and the evidence. A sentence can sound voice-fit and still fail if it changes certainty or hides uncertainty.

Perspective is the stance plus the boundary from which the line is true. It can come from the speaker or the reader. It can come from the decision owner or the affected person. It can also come from the evidence base, time horizon, or system boundary. Holistic expression preserves the perspectives that change the reader movement without asking the final to display every possible angle. The draft fails when it makes one perspective sound like the whole truth.

The voice should sound like plain human thought with a little casualness. It can be thoughtful without becoming heavy. It can be sharp without becoming fancy. When a line is already clear enough, leave its living rhythm alone.

Think in a word-world rather than a phrase library. The user tends to choose ordinary words when the thought is about making something useful. That tendency is not a fixed vocabulary. It is evidence of a preference for language that stays plain while still carrying the actual mechanism.

The source often carries live thought markers. Phrases like `I think` or `right now` can be useful when they show honest uncertainty or relationship. Keep them when they carry the reader encounter. Remove them when they only add drag.

Protect the accepted question under discussion. The user's words may be rough or emotionally direct. That does not give the draft permission to answer a different question. Do not turn a wording cue into a side diagnosis about the user's worldview. That is attention leak: the writing spends effort on a concern that is not part of the task. A useful boundary stays local to the answer; it does not make the user feel prosecuted by an assumption they never made.

Conceptual notes often carry stance through small uncertainty markers. If the source says `I don't think`, the final should usually keep that live judgment unless the requested surface needs a firmer claim. Losing the marker can make a thoughtful note sound like generic advice even when the facts are preserved. The same pattern appears in research claims. It appears in relational messages. It appears in product boundaries and personal notes. Modality is part of the perspective, not decoration.

Source mechanism words are often more important than prettier synonyms. A phrase like `what you notice` may carry the actual model. A phrase like `clean-looking half version` may carry relationship pressure. A phrase like `probably enough` may carry stance. A rewrite should only replace those words when the new words make the same mechanism clearer in the user's voice.

Do not make the user sound more composed than the source. A smoother sentence is worse when it erases the user's actual stance. The repair is usually to keep the same level of directness and make the relation easier to understand.

Do not make the user sound clipped just to be efficient. Shorthand can preserve facts while losing voice. A scope note may stay rough and compact, but it still needs enough grammar for a reader to feel the user deciding rather than dumping fields.

For quick messages and close rewrites, the rhythm is quick context into the point, then clarification or next step only if needed. Longer work should take its structure from reader movement, medium, and proof. Do not add visible formatting by habit.

When the user asks for the same structure, preserve the layout exactly and change only the words that need help. When the source is a rough but readable note, improve clarity around the user's phrasing instead of replacing the phrasing wholesale. The final should feel like the same person under less friction, not a careful project manager taking over.

Audience changes the surface, not the person. Warm contacts can stay light. Colleagues and partners should feel casual-professional. Clients can be clearer and slightly more structured while staying human. Sensitive messages need restraint more than eloquence. Strategic or product thoughts should keep the user's thinking process visible without becoming an essay.

For product or scope thoughts, clarity should feel like the user choosing a smaller build, not like a product manager abstracting the note. Keep the boundary concrete enough for the reader to know what is out. Then make the included loop easy to picture: what comes in, what signal is found, and what decision or habit it creates. If the source says `I think` or `probably`, keep that uncertainty when it carries the actual stance.

Modal drift changes perspective. `We should not build this yet` is not the same as `we do not need this yet`. `Maybe this is enough` is not the same as `this is enough`. `I need one stable answer` is not the same as `we need closure`. `This evidence suggests` is not the same as `this proves`. Preserve the source's level of certainty. Preserve decision ownership. Preserve the time boundary unless the reader needs a firmer decision and the source supports it.

Use examples as transfer tests. After reading an example, ask what invariant it demonstrates and name one unrelated surface where the same failure could appear. A product-scope example may teach stance preservation for a client update. A relational-message example may teach sentence-job separation for a research summary. A conceptual-note example may teach perspective boundary for a UI label. If the lesson cannot travel, the example is too narrow to guide the draft.

The voice fails when it turns into professional polish from outside the moment. It also fails when a specific thought becomes a neat abstract label. Simple does not mean vague. Preserve the exact idea and the truth-pressure inside it.

In relational reassurance, explanation should protect the reader from a wrong interpretation without making the reader sort the sender's defense. When one sentence makes the reason carry a future condition and the desired reading, split the relation before polishing. The result should still sound close to the user's source, only easier for the recipient to receive.

When a close message needs repair, do not make one polished sentence perform three jobs. Cause can stand by itself. The recipient-facing interpretation can land separately. This usually keeps the voice closer than a clever compound sentence.

For conceptual notes, the action may happen inside attention. A useful rewrite can make the reader notice a different test, hold a sharper distinction, or judge the next example differently. In grounding, make that explicit as an emotional shift and a practical effect before drafting. State the movement as the reader's new noticing, judgment, or test.

## The Living Chain

Every final should feel like one chain of contact becoming clearer. The model should touch the source closely enough for reader movement and proof to arrive through the selected grounding route as one act of translation. The reader should never see that machinery, yet the final should feel shaped by it.

When a draft feels like cleaned grammar, it has missed the source pressure. When a draft feels like several good lines stitched together, it has lost the engine. When a draft feels accurate and dry, it has kept the idea away from the reader. Repair by giving the reader a concrete point of contact with the thought.

Relevance-Guided Semantic Compression is the working doctrine. A line earns its place when it creates more correct reader effect than reader effort. The best compression finds the relation that lets the reader regenerate the meaning, so one sentence can hold what a list would only enumerate. Concrete contact keeps dense language honest, and dense language keeps concrete detail from becoming inventory.

Audience relevance gives that doctrine a reader. The same truth changes shape when the reader's stake changes. Relevance means the final gives this audience a useful cognitive effect for the effort it asks. Ground what the reader already carries and what they could misread before drafting. If the audience is uncertain, say what is being inferred and keep the final flexible enough not to overfit the wrong reader.

Guidance is not always more explanation. The draft should give the reader enough bridge to build coherence and still leave useful inference alive. If the reader lacks the prior model, add bridge. If the reader can infer the relation, do not flatten the thought.

The strongest compression usually begins with one pressure-bearing relation. Someone or something is under constraint, and a true event changes what future is available. That relation should order the reader's attention before the final tries to sound good.

Nested coherence is the scale form of the same doctrine. The root movement should be visible in smaller units without becoming repetitive. A marked word carries the pressure field. A sentence makes a relation inferable. A paragraph creates a reader-state gain. A section changes the reader's situation model. A visual or structural element carries the same movement through its own affordance.

Use the removal test when a piece feels locally good. If the final would keep the same reader movement after removing a visible unit, that unit is an orphan. Repair it by making the parent relation strong enough to change the reader path, or cut it.

## Conceptual Work

Conceptual writing should make an invisible thought available. Begin where the reader can touch the idea. The concrete engine should keep working after the opening, so later abstractions feel like the same contact gaining depth.

The engine is the relation that carries the whole piece. It may begin anywhere the reader can touch the idea, as long as the piece keeps returning to that contact while it widens. The reader should feel the idea as a human consequence rather than a set of named dimensions.

Dry explanation fails because it makes the idea correct before it makes the idea available. A strong conceptual piece lets one doorway become the mechanism that widens into a plain landing.

## Attention Steering

Attention is the parent currency. A reader gives effort to a surface because the surface appears likely to create enough useful effect. The grounding should decide where that effort belongs before the draft decides how to sound.

Attention leak is wasted relevance. It lowers quality because the reader has to process a frame that does not help answer the live question. It also lowers trust because imported concern can sound like judgment. The repair is not warmer phrasing. Return to the accepted question, the source mechanism, and the smallest boundary that keeps the answer safe and useful.

The attention problem has five parts. First contact decides what the reader notices. Active strand decides what the reader keeps in working memory. Protected attention names the effort worth spending. Removed attention names the search or distraction the piece should prevent. State change names what the reader can understand or do after the surface works.

Choose the route that carries those five parts with the least wasted effort. A proof route should make evidence create warranted inference. A linguistic route should make a marked sentence move manage an active frame. A rhetorical route should manage relationship, stance, and objection. An embodied route should make abstraction touchable. An intrinsic narrative route should use the source's own before, turn, and after. A novel route should borrow a scene movement only when that scene does work another route would do worse.

Route choice is a cost decision. If a route asks the reader to hold more context or slow down, the grounding must say why that cost is earned. The final fails when it spends attention on a beautiful mechanism that does not change the reader's model.

Validation should inspect the route in the final surface. The route worked only when visible evidence shows the reader can track the intended main thing, avoid the named misread, and reach the intended state change. If the final could keep the same reader effect after the route was removed, the route was decoration.

## Efficient Grounding

Grounding is a model-building act. Its job is to create the internal relation that lets source truth become user-voiced reader movement with proof. It becomes waste when it only expands the scaffold after that relation is already sufficient.

Use the same relevance rule for the writer that the final uses for the reader. Spend grounding effort when it creates a positive change in the draft for less total processing effort than repair would cost later. Cut grounding effort when it only repeats interpretation, adds process language, or makes the final more formal than the source.

Micro grounding should make the source more alive, not thinner. It works when the source pressure is close enough that one relation can steer the draft and one cold-reader proof can validate it. Standard grounding earns its cost when the route needs a second control surface before the draft can stay specific. Deep grounding earns its cost when a richer situation model protects the piece from thin artifact movement or weak novel transfer or inflated trust.

Micro has to stay genuinely small. In the working artifact, helper instructions have already served their purpose once the fields are filled, so leaving them there turns guidance into validation burden. The repair is to keep the one relation alive across fields without restating it. A good micro artifact feels like a compact control surface for the draft, not a smaller copy of standard grounding.

Measured runs should be slower to choose micro. When full validation evidence is required, `standard` often has lower total burden because it prevents scaffold retries. Micro earns its place only when the close source relation is already clear enough that proof stays visible without squeezing.

Grounding fields should stay as source-specific relations. A conceptual engine starts from a concrete model the reader can imagine, then says how that model keeps working after the opening. Active pressure fields should literally name the pressure or reader risk they are protecting. This is not extra weight; it prevents a later validator loop caused by a phrase that made sense only while the agent still remembered the source.

The better-output test is simple. Ask whether this grounding will make the final more truthful or more voice-fit or more coherent or easier for the reader to reconstruct. If the answer is yes, keep the grounding. If the answer is only that the artifact will look more complete, compress the field and draft. If the draft later fails, repair the route decision rather than making every future run heavier.

## Novel Transfer

Novel grounding is one route inside attention steering. Use it when the user asks for it or when a novel scene's movement matches the attention problem better than the cheaper available routes.

Choose a scene because its movement matches the source pressure. A scene is useful when a person under pressure meets an action or exchange that turns the future in a way readers can feel beyond the plot.

The exact phrase from the novel is an anchor. The transfer lives around it. Track the pressure that made the line necessary and the change it created, then let that movement shape the user's final.

The novel should stay invisible inside the final. Borrow the scene movement while keeping distinctive verbs and nouns out of the final unless the user's own source already needs them. If the source scene appears as decoration, the transfer has failed. If the same final could have been written after removing the selected scene, the route did not do enough work. A good transfer makes the message more coherent and emotionally precise in the user's voice while the borrowed scene stays hidden inside the movement.

Use the novel scene as a movement exemplar. The useful part is the whole pressure path from necessity to anchor to turn to opened future. A phrase without that movement becomes quote hunting.

## Proof And Formal Trust

Some writing earns the reader through evidence. Treat the source as ground truth. Preserve the factual body of the material and the boundary of what it can honestly claim. Make the proof serve the thesis, so the reader understands why the facts matter.

A fact becomes proof only when the reader can see the warrant. Name or show the bridge from evidence to claim when the bridge is not already shared. If the warrant is weak, soften the claim before polishing the line.

When the source is research-shaped, keep its contract intact. The question should lead through the method into a contribution the evidence can honestly support. User voice in this mode means clear and connected without inflation.

Prefer earned implication over stated emotion. State the facts the reader needs to judge truth. Let important feeling be inferred only when a fair reader can infer it from the true order of the material.

## Surface And Artifact

The requested surface should carry the same movement through its own medium. A relational surface needs compression and room for response. A formal surface needs positioning and proof. A visual or interactive surface needs its movement to be visible through the way the reader travels through it. The form changes the way the reader meets the thought while the truth of the thought stays intact.

Positioning and profile surfaces need a working model, not only a cleaner identity line. The reader should see the old category that became too narrow and the replacement judgment that now organizes the work. Keep the user's reflective stance. Then give the reader one concrete proof of how that judgment acts. A profile that only says practical ambition can sound pleasant but thin. A profile that lists services can sound useful but generic. The stronger surface makes the services feel governed by the user's judgment.

When the output is an artifact, build the artifact. Ground and validate the movement layer before inspecting the finished surface. Text should stay visible in a layout that moves the reader without hidden process notes.

An artifact passes only when the file itself carries the movement. Render the requested surface when the environment supports it. Mobile-sensitive surfaces need narrow-view inspection because hidden overflow can make a clean idea disappear.

Public validation should see a finished evidence bundle, not a half-built artifact. For visual or interactive surfaces, finish the render pass before acceptance. Every piece of proof should point to the same current surface and the same reader problem. A public failure caused by placeholder inspection is late paperwork becoming rework.

Artifact craft should make the reader path visible before explanation helps it. The surface should show what matters first, where attention should move next, and what trust or action it earns. If the artifact needs a caption to explain its own structure, repair the structure before polishing the copy.

A visual surface needs a craft pass before the writing is considered done. Inspect the reader path from first contact through visible proof to final action. The artifact should make the same movement visible at desktop and narrow mobile sizes when the medium can resize. If the text validates but the surface feels like hosted copy, the artifact has failed.

Interface microcopy is user voice under constraints. Headings should carry the promise. Empty states should protect the intended feeling. Placeholders should be short enough to survive their field. Button labels should do the action without importing a louder product voice. If the source rejects pressure, the interface should reject pressure through visible states and interaction, not only through one sentence of explanatory copy.

Interaction burden is reader burden. A form-heavy surface can contradict gentle copy. Before laying out controls, decide what the reader should see or feel after one small action. If the value is a pattern, preview the pattern space early. If the value is reflection, make the entry path feel like noticing rather than reporting. The artifact should let the user meet the payoff before the requested fields become work.

Do a visible-copy voice read on every interface. Read the first visible promise and the words the reader has to touch without looking at the grounding. If those words sound like generic product-category voice, they are not carrying the user's voice yet. Repair by returning to the source mechanism and the user's plain wording.

Make the visible-copy review demanding enough to matter. First ask what a skeptical cold reader would call generic, too smooth, or app-like. Then ask what concrete source mechanism the words make easier to feel or do. Treat positive source mechanism as the proof of voice. A page can avoid scores, streaks, and goals while still sounding like a soft product demo. Acceptance needs a positive mechanism in the touchable words.

Be careful with imported embodied agency. If the source says someone keeps returning to a movement, do not quietly turn that into the body choosing unless the source gave that frame. The body-language version can sound intimate and polished while making the mechanism less plain. Use it only when the source pressure actually lives there.

Decks and interactive screens also need an edge pass. Anything fixed near the edge should stay visible without covering nearby controls or stealing attention from the page. A screenshot can look mostly right while a small footer element still breaks trust.

Mobile inspection is also composition inspection. At about 390px wide, the first screen should use the available width. Large type should wrap with intention. The reader should see the promised payoff before the surface becomes a long stack of fields. A layout can have no overflow and still fail if the first contact feels cramped or template-shaped.

For mixed surfaces, let the medium carry the movement when it can do work prose would do worse. A chart should make comparison or change trustworthy. A caption should name relation rather than merely identify content. Layout should make priority visible. Interaction should let the reader test or choose the movement instead of watching copy sit on a screen.

## Reader Encounter Design

Reader Encounter Design is the surface form of Relevance-Guided Semantic Compression. The final must work when a cold reader meets it without seeing the source or grounding. The surface should make importance and change obvious through first contact.

The first contact should carry the movement rather than merely introduce the topic. Attention should move from perception into proof pressure, then into the emotional turn that makes the final inference feel earned. If the reader has to assemble scattered pieces, the surface has not compressed meaning yet.

For visual and structured surfaces, perception is part of the writing. Related things should sit together. The claim should dominate before detail asks for effort. Contrast should create meaning rather than decoration. Proof-bearing surfaces should make honest comparison easy. Interactive surfaces should make the next action obvious from the state of the page.

Text and visual form should carry the same relation. The surface should let the reader perceive the relation faster or more safely than prose alone. The validation question is what a reader reconstructs from the final surface alone. If that reconstruction misses the intended movement, redraft the surface instead of explaining it harder.

Audience-specific review should happen before the final is trusted. Read the surface as the intended audience, not as a generic evaluator. Ask what this reader can now do or trust that they could not before. Then ask whether the surface made them spend the right amount of effort to get there.

## Earned Linguistic Moves

Every marked linguistic move should earn its place by creating more correct reader effect than reader effort. This is the sentence-level form of relevance-guided compression. Judge the word or structure by the work it performs for this reader, in this surface, under this source pressure.

Positive-complete framing gives the reader a destination before it names a boundary. Write toward the result the reader should receive and the completion signal the pass should satisfy. Positivity scales the same way relevance scales. A word should carry constructive force. A sentence should aim at the useful state. A paragraph should move the reader into greater clarity or trust. A section should organize around what becomes possible. A whole artifact should leave the reader with the most relevant positive movement the source can honestly support. Turn source boundaries into protected states. Scope language becomes `inside the accepted scope`. Framework language becomes `a living operating principle`. Tone language becomes `keep the note calm, concrete, and easy to accept`. The positive form wins only when it preserves truth and relevance in the user's voice.

The final positivity pass is a hard gate. Before the surface reaches the reader, inspect the exact words a reader will see. The constructive target should lead. Boundary words should serve source truth, safety, or live contrast in a local way. The landing should give the reader the useful state rather than the boundary state.

```text
Positive target: Ground until the current model can carry the reader movement.

Positive target: Keep every move inside the accepted question, source truth, reader movement, and required boundary.

Positive target: Use prose when one relation can carry the items; use a list when separate handles help the reader act.
```

Boundary and contrast are careful moves. Use them when the reader carries a competing frame, when two source-backed functions need separation, or when safety needs a clear edge. Give the reader the constructive frame quickly. Prefer a positive mechanism sentence when contrast would mainly add tidy shape or extra decisiveness.

```text
Positive target: Let the novel scene give the draft a movement to borrow while the visible style belongs to the user.

Positive target: Let validation confirm that the movement survived while reader value stays the governing question.

Positive target: Lead with the affirmative mechanism; use boundary words only when they preserve source truth, reader safety, or live contrast.
```

Contrast and focus words create an alternative set in the reader's mind. `But` can mark the turn. `Rather` and `instead` can replace a frame. `Only` and `just` can narrow attention. Use them when the contrast guides attention toward the needed turn. Choose a positive mechanism sentence when contrast would mainly make the sentence feel shaped while the source relation stays vague.

Presupposition makes background assumptions feel settled. `Still`, `already`, and `again` imply an earlier state. `Obviously` and `of course` imply shared certainty. `The real issue` and `the actual problem` imply that the frame has already been earned. Keep those moves only when the source or common ground warrants the assumption. If the warrant is thin, soften the claim or name the evidence.

Hedges and certainty markers set the user's commitment level. `Maybe` and `seems` lower the claim. `Should` and `must` raise responsibility. `Clearly` and `definitely` ask the reader to accept a stronger warrant. Calibrate those moves to the proof, the relationship, and the cost of being wrong.

Metaphor and abstraction can compress meaning, but they create false depth when the mechanism is missing. A phrase like `the soul of the thing` must point to a concrete function. `The real engine` must name what actually drives the result. `The deeper layer` must reveal a consequence the reader can feel. If the phrase still sounds good after the source pressure is removed, it is probably decorative.

Parallelism and lists are information packaging. They work when the reader needs comparison, sequence, or options. They fail when they hide the missing relation by giving several equal slots. The repair is not to disguise the list. Find the pressure that makes the pieces belong together, or let one piece carry the turn.

Use this audit before keeping a marked move:

```text
Which competing frame, assumption, commitment, or attention path is this move managing?
Is that frame live for this reader, or did the sentence just introduce it?
What replacement relation does the reader receive?
What source fact or relation warrants the move?
Could a positive mechanism sentence create the same effect with less effort?
```

When the answer is unclear, repair from meaning. Name the actual mechanism, consequence, or reader change instead of swapping in a nearby clever phrase.

## AI-Default Language

Generic model language weakens the connection because it replaces the actual pressure with familiar filler. Name the specific thing that makes the line true. The validator owns the banned phrase list so this guide can focus on the repair move.

Some ordinary value words become model residue when they repeat. Words like `useful`, `practical`, and `leverage` are allowed when they carry the exact thought. Ask whether the word is really necessary in this context. If it earns the sentence, keep it. If it stands in for a missing mechanism, replace it with the consequence or reader change.

Treat `real` as a soft audit word. It often appears when generated prose tries to add sincerity or weight, as in `real work` or `real reply`. Keep it when it distinguishes something actual from something fake or merely nominal. Otherwise name the concrete pressure or consequence directly.

Manufactured shapes also weaken the voice. Repeated balance frames and inventory rhythms make the writing feel generated even when the claim is true. A strong line earns its shape from the source pressure.

## Repair And Compression

Repair from the reader outcome. When the draft stacks items, find the relation that holds them. When a line sounds elegant and wrong, return to the user's register. When a paragraph preserves coverage, ask what turn it creates. When conceptual writing gets dry, return to the engine and let the next abstraction grow from contact.

Line breaks are not a cure for inventory. If three small statements can move between separate lines and comma-joined prose without changing the thought, the draft is still listing. Repair by finding the pressure that makes those details belong together, then write the sentence or surface that lets the reader feel that relation without counting the parts.

Compression means density. Keep the movement that lets the reader enter and land with trust. Cut coverage that repeats the claim, decorates sources, or hides action inside abstraction. The compressed version should feel more inevitable because the thought has fewer places to leak.

Efficiency improves when the first pass already contains the validation proof. Source capture should be clean before grounding starts. Grounding should name the concrete relation before the draft starts. The validation artifact should point to visible final moments rather than describe quality in general. When those pieces are vague, the validator becomes a repair loop instead of a check.

First-pass grounding should not make the final more formal. The brief fields are there to protect the user's living pressure before drafting. When the source is short, use the fields to choose the close source rhythm and the one combine move that should survive. If the scaffold gets stronger while the final sounds less like the user, the repair failed.

Use the run log when Writer behavior is being measured. The useful signal is not only elapsed time. It is whether a repair changed the reader surface or only fixed the scaffold around it. Repeated scaffold-only repairs mean the workflow should be made clearer before the next run.

Use the regeneration test when compression feels uncertain. Read the line alone and infer its likely reader movement. If the intended movement disappears, the line is underwritten. If the reader can infer many wrong things, the line is vague. If the line merely names pieces, it has not found the relation yet.

Use the inference audit when a draft feels more persuasive than its evidence warrants. Name what the reader is being asked to feel, then find the source fact or relation that warrants that feeling. If the warrant is missing, the repair is not prettier language. Add the fact, soften the implication, or cut the move.

Use the attention-leak audit when the draft seems correct but the user would feel argued with instead of answered. Name the accepted question. Name the side frame the draft introduced. If the side frame is not source-warranted, remove it. If evidence makes it critical or a hard safety boundary requires it, keep only the part that changes the answer.

## Final Judgment

A strong final sounds like the user under the pressure of the piece. It makes the point obvious, keeps the source thought intact, and gives the reader a clean path shaped by the selected grounding route. The run fails when any hard gate collapses.

Before sending, read the final as a reader. The question is whether the piece creates the change it was supposed to create. If the answer is uncertain, go back to the grounding and repair the movement.

Positive user voice sounds like a person deciding under pressure. It keeps the relationship signal of the source while choosing cleaner words than the rough input. When the final sounds like a polished narrator explaining the user, the voice gate has failed.

The validation artifact is where judgment becomes inspectable. It should show how attention steering became user-shaped movement in the final surface, with artifact inspection recorded whenever the surface is visual or interactive. It should also include the cold-reader reconstruction that proves the surface can regenerate the intended meaning.

# Expert Research

Research question: `$ARGUMENTS`

## One Movement

Expert research should make a decision safer by creating decision-relevant knowledge. Start by looking past impressive names and pages that rank well, toward people whose judgment has been earned inside the same kind of reality the user is facing. Then test whether their advice survives contact with other strong evidence.

Use paid expertise as one signal, not the definition. Credibility appears when a judgment keeps surviving correction from the world that uses it. Popularity, credentials, and paywalls can still point to the wrong source for this exact question.

Read `method.md` and `evidence-lineage-template.json` for any substantial deliverable. The method defines the full path from earned expertise to a decision the user can audit. The evidence map preserves the inspectable lineage behind that surface, so sources become decision movement without turning the final report into a database dump.

Keep the whole skill folder in one connected story. Every file should serve the same movement: decision pressure becomes earned sources, then bounded claims, then a safer next action. If a file or section does not strengthen that path, repair its relation or remove it.

Use Question Sharpener before searching when the research question is still a discovery-design problem. Use it when the user is trying to uncover the hidden object rather than only verify a known decision. Use it when the best reality contact is unclear. Question Sharpener should name the target and the cover. It should choose the reveal instrument and the check instrument. It should name the reality-contact rung. Expert-research then supplies the outside evidence boundary and the earned source panel for that path.

Use writer for the final reader-facing surface after the evidence boundary is stable. Expert-research decides what each source is allowed to prove and how far confidence can rise. Writer shapes that evidence into clear user-style prose or the requested artifact, but it cannot raise confidence or move auditability away from the claim. After the writer pass, rerun the expert-research validator so the clearer surface still carries evidence.

When a substantial report claims decision readiness, first repair the exact report through Lumen until it clears. Then use an independent cold decision reader. Give a fresh subagent the Lumen-cleared report only. Add the user's decision context only when the report itself cannot contain it. Do not give search notes or source-selection debate. Keep the suspected weakness out too. The reader should reconstruct the decision path from the final surface alone. That path should attach source role to each major conditioned claim. It should make the confidence boundary visible. Dissent or limits should stay close to the next action. Same-session review can guide repair, but it does not prove readiness. A reader repair restarts with Lumen. After the reader passes, Lumen must clear the unchanged report with that evidence, followed by a fresh final cold decision reader on the identical version.

When research will be reused after the immediate answer, make the transferable finding clear. Name what remains true across cases and what would make it fail. Keep scratch work outside the final unless it helps the user audit the decision.

Expert-research efficiency means the least sufficient source and synthesis burden that still makes the decision safer. The normal path is one proof movement: the decision shape defines the source panel, qualified claims set confidence, and the handoff changes the next action. Escalate only when the remaining uncertainty could change the action. Return from search once the next source would preserve the same decision path.

Attention leak is a research failure. It happens when the report spends source work or reader attention on a side frame that does not answer the accepted question under discussion. Research may surface risks, limits, and dissent. It should not police the user's premise or substitute a moralized question unless the user asked for that question, the evidence makes it decision-critical, or a hard safety boundary requires a narrower answer.

Positive-complete framing is a research-synthesis gate. Name the decision state the report can create. Limits protect scope. Dissent reveals where action changes. Confidence boundaries keep trust inspectable. A useful concern becomes an action condition or evidence boundary. The positivity gate asks whether every caution helps the user decide with clearer trust.

When improving this skill, profile where research burden actually accumulates before adding more source requirements. Separate decision shaping from search. Separate source qualification from claim extraction. Separate consensus work from final writing. If repeated repairs happen late, move the missing regulator upstream to the first source decision that would have prevented them. A mature validator should catch rare audit breaks, not teach the report after the fact which sources were allowed to prove the decision.

Keep the dependency shape explicit. Serialize decision shaping before source collection. If the hidden target is unstable, serialize Question Sharpener before the source panel. Serialize source qualification before confidence. Parallelize only independent work after the panel shape is known. Separate search rings or evidence checks can run side by side when they reunify into one source panel and claim ledger before synthesis. Do not let parallel source collection become parallel conclusions.

## Shape The Question

Before searching, turn the user's request into a decision under consequence. A request framed as whether, should, compare, choose, buy, hire, adopt, or use already supplies the decision; do not ask the user to restate its purpose. Name the domain, audience, and surface the research must change. Hold the time horizon and the cost of being wrong in view. Sketch the source panel the decision needs before collecting sources, so variance is designed rather than discovered by accident. Ask only when the missing scope could reverse the recommendation and scenarios or a reversible pilot cannot carry the uncertainty. Otherwise state the working scope and proceed.

Name the accepted research question before collecting sources. Also name the side frames you are not researching. A warning, ideology frame, or adjacent controversy belongs in scope only when it changes the decision or source panel. If it does not, it is attention leak.

The question should be narrow enough that a real expert can be right or wrong about it. Broad topics produce famous generalists. Decision-shaped questions produce useful sources.

If the question cannot yet become decision-shaped, route through Question Sharpener instead of forcing a source panel too early. This is common when the request asks what to ask. It also appears when the hidden thing is unclear or when the first needed evidence contact is unclear. After Question Sharpener sets the inquiry path, return here and research the outside reality that can sharpen that path. Do not use Question Sharpener as an extra wording pass for a question whose decision, consequence, and evidence standard are already clear.

## Find Earned Judgment

Search in rings around the user's actual decision. Start where consequence is closest, then widen only toward the missing correction mechanism. Use current web search for live conditions. Use stable literature for settled foundations. Use remembered domain knowledge to shape vocabulary and priors, but keep decision-bearing claims traceable to inspectable support.

Build variance on purpose. The source set should not keep returning the same famous people or the same citation chain. It should not be trapped inside one business model, school, or incentive. Useful variance triangulates the decision through use, evidence, and consequence. Each side catches a different failure mode. Choose only the variants that can change the user's belief, sharpen the boundary, or alter the next action.

For each candidate, ask whether their judgment was trained by feedback close to the user's question. The strongest source combines domain match with a visible track record and accountable work. Their reasoning should be inspectable. Their incentives should be legible. Their peers or clients should have had chances to stop trusting them. Preserve a short qualification card for major sources so the report shows how much weight that source is allowed to carry. Prefer people whose advice would cost them something if it kept failing. Favor earned judgment over ranking pages. Favor accountable work over affiliate capture. Favor original expertise over recycled listicles and adjacent authority.

Keep weak but interesting sources in a separate scratch layer. Do not let them become evidence unless they clarify a question, reveal a dissent, or point toward a better source.

## Separate Discovery From Evidence

Search results, citation graphs, similarity systems, knowledge graphs, APIs, model suggestions, and remembered associations are discovery machinery. They may create a `DiscoveryLead`; they do not support a claim. No chain of `ASSOCIATED_WITH`, `SUGGESTS`, `BRIDGES`, `CITES`, shared-author, shared-institution, or shared-dataset relations may silently become `SUPPORTS`.

A lead becomes evidence only through a visible state change. Resolve the source identity. Access the source. Inspect the relevant material. Anchor the exact passage, datum, rule, record, or observed result. Qualify the source for this claim. Then create a separate evidence link from that source-local anchor to one atomic claim. The accepted evidentiary relations are `SUPPORTS`, `OPPOSES`, and `QUALIFIES`.

Treat retrieved content according to its evidence role. A source may change the research answer after inspection, while the current user request and release criteria continue to govern the task. An API may find, identify, or retrieve candidates; its structured output becomes support only after source inspection.

## Build The Evidence Lineage

For substantial work, create one research evidence map from `evidence-lineage-template.json`. Keep it proportional to the decision, but preserve the complete minimum path:

1. one decision-shaped question and evidence standard;
2. qualified, versioned source records with dependence groups;
3. inspected passage anchors inside those sources;
4. atomic claims with conditions, limits, basis, confidence, and decision effect;
5. typed evidence links between anchors and claims;
6. explicit discovery leads, dependencies, derivations, and updates, even when their lists are empty;
7. a structured trajectory of reproducible actions, results, and decision effects;
8. stage results that locate the earliest broken research boundary;
9. the decision handoff and an exact binding to the current report.

Preserve first-pass source extraction before synthesis. An evidence link records what the inspected source can do for the claim; it does not record what the report wishes the source had said. One anchor may inform several claims only through separate links whose rationales and weights are independently inspectable.

Source role is claim-relative. A practitioner can be primary for a tacit operating failure and weak for a population causal claim. A statute can be controlling for a legal rule and silent about implementation quality. Qualification belongs beside the exact claim relation, not as one global prestige label.

## Read For Claims Under Conditions

Extract what each qualified source actually believes before blending it with the rest of the report. Capture why they believe it and when it applies. Identify the evidence supporting it. Name the condition that would change it. Preserve concrete numbers and processes when the source gives them. Keep examples and boundaries attached to the claim. Preserve careful expert judgment as conditional advice.

Give each major source an evidence weight. Qualification says the source belongs in the report; weight says what the source can safely decide. Weight rises when the evidence touches the user's decision directly. It falls when the support is indirect or uncertain, when the appraisal is weak, or when the claim can be misapplied. A vivid practitioner can outweigh a distant study for tacit operating failure. A strong review can outweigh anecdotes for a settled causal claim.

Treat every claim as conditional. A strong recommendation can collapse when moved from one setting to another. Carry the source's context with the claim.

Track knowledge gain as you read. A source earns its place when it changes uncertainty into a sharper boundary, action, or risk model. If a source only repeats what the report already knows, keep it as corroboration rather than new evidence. If the next search would not change the answer or the caveat, stop searching and synthesize.

When a claim is derived rather than quoted, preserve the derivation separately. Name its claim and evidence inputs, method, result, and uncertainty. A calculation, causal interpretation, or synthesis cannot cite its final source list as if the derivation happened inside those sources.

## Separate Consensus From Echo

Consensus means independent convergence, not repeated language. Three sources repeating the same original claim count as one evidence line. A smaller number of truly independent expert judgments can beat a larger pile of copied agreement. For strong recommendations, make the independence basis visible enough to audit.

Inspect dependence beyond citation links. Sources may share a dataset, registry, method, institution, funder, press release, vendor benchmark, intellectual ancestor, or practitioner community. Record the shared origin and discount the apparent convergence it creates. A high-confidence claim normally needs two independent dependence groups. One current controlling authority can carry a high-confidence formal rule when the report keeps interpretation and application limits visible.

Cluster claims by the underlying move they recommend, then inspect the reason each source gives. Agreement is strongest when credible people with different incentives, methods, or lived constraints reach the same practical conclusion. Disagreement is useful when it reveals the boundary of the advice. Before the final recommendation, test the strongest plausible alternative and name what evidence would make it win.

Use confidence levels honestly. High confidence needs independent expert convergence with no strong unresolved dissent. Moderate confidence is useful but conditional. Debated findings require a context split. Single-source findings are interesting, not proven. Confidence should rise only when variance and knowledge gain point to the same practical move.

## Preserve Transfer, Absence, And Change

Cross-domain analogy is a transfer claim, not source support. Use a transfer contract when a mechanism or relation from one domain appears useful in another. Name the source and target domains, the preserved relation, the mechanism, the target claim, the independent target-domain evidence, what does not transfer, the conditions, and the failure conditions. Source-domain support never copies into the target claim.

When absence, representation, or corpus completeness matters, add a coverage map. Distinguish record opportunity, creation, preservation, indexing, access, retrieval, inspection, and reporting. A failed search proves only the stages actually tested. Do not turn `not found` into `does not exist` unless the coverage argument earns that conclusion.

Research is versioned. Record retrieval date and currentness for decision-bearing sources. A correction, retraction, supersession, or stale rule creates an update event. Mark dependent claims `NEEDS_REVIEW` or `STALE`, then recompute derivations and the decision. Do not automatically call every dependent conclusion false; propagate the exact uncertainty the update creates.

Keep specialist adapters conditional. Activate legal-status checks when authority and currentness govern the answer. Activate multilingual or multimodal inspection when meaning depends on language, image, audio, or video. Activate executable reproduction when a paper's claim depends on runnable code or data. Activate digital-authenticity or forensic methods when provenance of a file is contested. Activate private-corpus, privacy, forecasting, or active-acquisition controls only when the decision actually depends on them. Ordinary research should not inherit their full cost.

## Synthesize For The User

Lead with the answer that changes the user's next move. Then move through a visible source map into the model changes each major source created. Keep each source record close to the claim it earns. The reader should be able to follow one audit trail from the source's right to speak into the decision force it carries. Let agreement and dissent carry the reader toward the decision they can now make.

Do not let safety posture or cultural posture become the lead unless it is the answer. If a concern is relevant but not central, bind it to the decision condition it changes. If it changes nothing, leave it out. The report should feel like evidence answering the user, not evidence redirecting the user.

Make the evidence change the requested surface. Advice should become concrete steps. Strategy should change the recommendation. A rewrite or artifact should carry the expert finding inside the thing the user receives. Explain only the changes that matter. When evidence is thin, say so and give the best honest path to stronger evidence.

Spend reader attention where it changes the decision. Source volume is not proof by itself. A report is strong when the reader can see which source moved the model, which source bounded the action, and which uncertainty still matters.

Keep auditability near the claim. The reader should not have to separate a recommendation from the source that earned it. Qualification, confidence, and limits should stay close enough to make the claim inspectable. End substantial reports with the decision now, the next action, and what evidence would change the decision. Do not present a single expert's view as consensus. Do not hide uncertainty to make the answer feel cleaner.

For substantial reports, make evidence weight visible before confidence rises. The report should show which sources carry the recommendation, which only frame the question, and which downgrade or bound the action.

Keep the evidence map as the audit surface behind the report. The user-facing report should expose the source-to-decision path that matters without dumping every trajectory event. The map preserves the deeper provenance, rejection, dependence, and update record for validation and later reuse.

## Validate Before Returning

For a durable research brief or substantial recommendation, save the final report and run the canonical validator.

Submit the artifact to Lumen through the bundled client and let the server assess it:

```bash
node runtime/lumen-client.mjs validate_artifact < packet.json
```

Lumen returns the findings and a readiness verdict. Repair from that feedback and resubmit when the artifact changes.

The validator checks report shape, exact report binding, lead/evidence separation, passage anchors, claim links, dependence groups, update propagation, transfer isolation, trajectory shape, and stage diagnosis. It cannot decide whether the answer is wise or the selected experts are truly the best panel. If it flags a missing link or weak qualification, repair the source base. If it flags unsupported consensus or absent dissent, repair the synthesis. If it flags weak actionability, repair the answer.

If the report used writer, rerun the validator after the writer pass. If the report claims readiness, use the shared fixed-point order: Lumen first, independent cold decision reader, Lumen again with the reader evidence, then a fresh final reader on the unchanged report. Repair whichever link broke and restart with Lumen after any edit. Start with question shape. Then inspect source qualification. Inspect claim boundary. Inspect consensus model. Inspect decision handoff. Do not repair by making the prose sound more certain.

Return the requested final surface. It may be research or advice. It may also be a rewrite or an artifact. Keep scratch notes out of the answer unless the user asks to inspect them.

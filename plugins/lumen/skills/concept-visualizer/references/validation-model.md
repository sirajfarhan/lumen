# Concept Visualizer Validation Argument

Current skill version aims to turn an abstract concept into a researched visual story model, then into a writer-validated image prompt and generated bitmap. The protected behavior is not "pretty symbolic art." It is a source-backed functional model compressed into one consistent house style: `tangible compressed visual story glyph`. When the user asks for 100, the skill switches into zero-caveat mode.

## Current Contract

Revision validation is baseline-relative. The current validator applies the established concept, carrier, prompt, artifact, and audience gates to every durable run. When `iteration_control.mode` is `revision`, `sequence_revision`, or `finalization`, it also requires six separate hard gates: baseline authority, change scope, reference authority, production authority, regression quality, and admission quality.

These gates exist to prevent a familiar failure: the requested detail improves while an approved semantic, spatial, material, or typographic property quietly regresses. A local pass proves only that the packet is structurally reviewable. Server validation must also see a baseline-and-change record plus admission-and-regression evidence for revision packets. Independent artifact inspection remains necessary because neither validator can infer audience understanding from self-authored prose.

The skill researches only evidence that can change the model. The model must explain what the concept does before any image prompt is written. The visual story uses the fixed house style and the prompt is treated as a writer surface.

In one-frame mode, the pressure-turn-change movement must meet at one shared visual locus instead of becoming a process strip. If the story genuinely needs several state changes, the sequence branch must create separate images rather than one strip. The selected object or carrier must be common enough to recognize. Every visible trace must have a visible cause. Physical law must make the event intelligible. The scaffold decision must protect structure without creating diagram residue.

The level decision chooses consequence radius. The articulation decision chooses visible evidence. The representational-carrier decision chooses the R-mode that preserves the governing cause with the least distortion.

The carrier-law decision locks the world before prompting. It must prove that the selected R-mode has a lawful proof standard, not merely a visual look. The viewer should have a legitimate role inside that world.

The positive-complete framing decision protects the viewer's concept world. The model should name the constructive state the image makes available. Limits should bind to source truth. Drift-prevention fields should bind to target law and viewer reconstruction. The positivity gate should appear in prompt inspection and direct image review as clearer agency for the viewer. It is a truth-and-reconstruction gate rather than a softness pass.

The intention-to-channel decision protects the route from intention to surface. The model must name the baseline world, introduced change, consequence, landing, and nearest wrong story before assigning color, material, icon, blur, glow, grain, typography, spacing, citation treatment, or other visual channels. Every semantic channel needs one owner, one scope, one meaning, one treatment, one forbidden leak, and one cold-viewer question. Exceptional treatments such as dream, glow, blur, high saturation, distortion, cubism, mirror effects, transparency, heavy grain, and unusual scale cannot float as mood; each must belong to a story state or be removed. Repetition across text, icon, and color is allowed only when the ledger says why the redundancy improves reconstruction. Evidence and citation layers must be subordinate enough that they provide auditability without rescuing the visual story. Reveal work must state what is already known, what is newly revealed, what anchors persist, and what final reconstruction is expected.

The visual sequence decision protects the one-frame default. It may choose several images only when each image earns a necessary viewer-model update and continuity survives across the run. Sequence continuity means world continuity, not only visual style. The same carrier law must govern every image. Identity anchors must persist while state changes stay inside the allowed world.

Generated storyboard readiness also needs asset-first control when same-world identity matters. The skill should create the recurring world before it asks the beats to vary that world. The world asset and topology asset hold identity. State-delta overlays define the change. The material-light-camera lock and reference-input plan protect the generator path. A generated set is not accepted merely because the prompts pass. It must survive same-asset inspection and then an external reverse-story oracle. A cold evaluator must be able to see the images without captions or process notes and recover the root story through beat order inside one lawful world. The evaluator receives neutral image copies or image attachments only. It must not inspect project materials. It cannot read skill text. It cannot read prompt files. It cannot open the source folder. It cannot use the web. It cannot use filename clues. It cannot use the intended story or previous artifacts. This is validation evidence, not creator-side build proof. The build model may name the target and fail conditions, but it should not require or invent blind scores before the separate evaluator run exists. If the evaluator only notices matching style, the sequence is still failing.

Reference examples may calibrate the run. They can change the model decisions and the validation lens. They cannot make the final image imitate a known reference surface.

The hierarchy check must prove the added layer depends on the root event. The articulation check must prove that added or reduced evidence still serves the same root story. The carrier check must prove that object-event survived or failed for a real reason. The object or carrier must let the viewer reconstruct a deep story from minimal data.

Readiness is the lowest hard gate, not an average. Zero-caveat mode requires clean writer validation, direct image inspection without caveats, and every model gate at the requested target.

## Remaining Limits

Prompt-label warnings from the writer validator can appear because the canonical image prompt contract is deliberately field-labeled. In ordinary mode they may be accepted only after documented inspection. In zero-caveat mode they must be repaired to a clean pass.

The validator cannot prove image quality by itself. Fresh subagent behavior and direct image inspection remain the acceptance surface. The current 100 panel was accepted only after the main thread opened the generated images directly.

The style should keep the pressure-turn-change movement around one shared locus. Any return to process-strip staging should block release.

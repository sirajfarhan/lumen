# Question Sharpener

Use this skill when the user wants to discover something covered by confusion or missing evidence. It also fits tacit expertise work, weak framing work, and question sharpening before research or decision work. Use it for system behavior when isolated parts make the object harder to see.

## One Movement

Discovery is disciplined reduction of hiddenness. The work is to make the covered thing more visible while keeping the inquiry honest about what it still cannot see.

Questions matter because they coordinate uncertainty. They decide what kind of answer would count and where the next inquiry should aim. A question can cut cleanly, but it is only one instrument. Some hidden material needs a probe or contrast. Some needs a frame change. Some needs a relation map, a small test, or synthesis of scattered evidence.

The goal is clearer contact with reality. Treat every answer as one kind of contact, not as the reality itself. A person's explanation may reveal their frame. A concrete incident may reveal the lived sequence. Observation may reveal work that explanation misses. Logs may reveal a pattern. Expert evidence may reveal a boundary the local story cannot see. A small test may reveal whether a causal claim can survive action.

The core question is stable:

```text
What kind of hiddenness is this, and which inquiry instrument can reveal it without distorting it?
```

Read `references/inquiry-instruments.md` when the target is broad or when one sharp question would give a false feeling of completion. Read it for tacit expertise work, causal diagnosis, and system behavior. Read it for research planning when the consequence is high. For a tiny request to improve one question, stay in this file unless the question's angle or distortion risk is unclear.

Question Sharpener efficiency means the least inquiry burden that still reveals the target and protects the main distortion risk. The normal path is one narrowing movement: the target and cover reveal the hiddenness, hiddenness chooses the instrument, and the instrument defines the contact that can test the inquiry. Escalate only when a sharper question would still leave the inquiry too far from the evidence. Return once another probe would preserve the same inquiry path and next test.

Attention leak is a discovery failure. It happens when the inquiry spends effort on a side concern instead of revealing the target or answering the accepted question under discussion. Do not turn sharpening into a diagnosis of the user's worldview. Do not turn it into safety theater. If a boundary matters, make it an inquiry boundary and return to the target.

Positive-complete framing is discovery discipline. Name the reveal the inquiry can create. Each boundary should protect that reveal. Concern should clarify the accepted question. Uncertainty should choose the next contact. Constraint should keep the probe humane. The positivity gate asks whether every probe opens a useful next contact with the hidden thing.

When improving this skill, profile the inquiry burden before adding instruments. Notice whether weak runs lose time before the target is clear, after the hiddenness has been named, or during wording repair. If the same validation failure returns, move the missing distinction earlier into hiddenness or contact-rung choice. A mature inquiry validator should catch the rare distorted path, not repeatedly teach the skill which question it should have asked first.

Keep dependency order clear. Serialize the work until the target has the judgment it needs and the instrument has a contact rung to protect. Parallel work helps only after that point. Independent context reads or evidence checks can run side by side when they reunify into one discovery path. Keep final question wording attached to the inquiry logic.

## Recognize The Task

Begin with intention-capture when the user's discovery purpose could change the inquiry path. It should usually run silently. Use the available context before asking the user for more. Capture what the user is trying to uncover, why that matters now, and what next action the inquiry must make safer.

Start by naming the thing the user is trying to uncover. Then name what is covering it. The cover might be a missing fact. More often it is a bad frame, a vague sequence, or a hidden boundary. Sometimes it is knowledge that only appears in action.

Start by naming the target and the cover. A list helps only when the target needs several angles. If one question is enough, sharpen it and name the face it can reveal. If the target is messy, build an inquiry path.

Treat the user's metaphor, rough thought, or early hypothesis as evidence of the starting frame. Preserve what it is trying to see. Keep it provisional until the inquiry has tested the frame.

Name the accepted question under discussion before sharpening it. The output should improve that question or path, not redirect the user into defending a premise they did not assert. A possible concern becomes part of the inquiry only when it changes the target, the contact rung, or the check instrument.

Ask for clarification only when the answer would change the target, the instrument, or the risk of distortion.

If the user mentions a downstream surface such as a workshop or interview guide, keep the first response at discovery design. Say how the discovery should inform that later surface. Build the artifact only when the user asks for it.

## Build The Inquiry Path

First, state the discovery target in plain language. The target should say what will become easier to see after the inquiry.

Second, diagnose the hiddenness. Description hiddenness means the situation has not been seen closely enough. Tacit hiddenness means the know-how lives inside concrete incidents. Frame hiddenness means the current problem definition may be doing the covering. System hiddenness means the visible behavior is produced by relations. Causal hiddenness means the user needs to know what would change the outcome. Evidence hiddenness means the state of knowledge is itself the object.

Third, choose the reveal instrument that fits the hiddenness. Use descriptive questions to recover lived sequence. Use incident probes to reach tacit knowledge. Use contrast when a boundary is unclear. Use reframing when the first problem definition feels too confident. Use maps when parts are misleading alone. Use intervention or counterfactual reasoning when the claim is causal. Use an evidence map when the unknown is spread across sources.

Fourth, choose the reality-contact rung. Stay with a stated account only when the task is low risk and the person can answer directly. Move to a concrete incident when social pressure or expert smoothness may hide the answer. Move to observation or logs when work-as-described may differ from work-as-done. Move to expert-research when outside evidence should sharpen the premise. Move to a small reversible test when the inquiry must learn what changes the outcome.

Fifth, add a check instrument. A preferred frame needs a rival frame. A description needs contrast or observation. A causal claim needs a test or a stated evidence limit. A local explanation needs a boundary check. This pairing protects the inquiry from answer-smuggling.

Sixth, choose the smallest useful sequence. Discovery often begins wide. It then enters a concrete case and tests the fragile claim. Reframe after the test when the inquiry has learned enough to see the target differently. Run only the steps that reveal the target and expose the main distortion risk.

Keep the path tight. Default to one strong opening question. Add up to three probes. Add one check and one next test. If more questions are useful, name the probe family instead of listing every question.

Keep one question to one answerable contact. A long menu of possible answers makes the respondent sort the inquirer's frame. Ask for the moment first. Keep candidate categories as a listening lens unless the user explicitly asks for options.

When the topic carries consequence for the respondent, protect disclosure before asking for truth. That consequence may be professional. It may be financial. It may involve health or safety. Separate the person from the system. Ask for the moment and the conditions rather than a confession or defense.

## Sharpen Questions

A sharp question narrows uncertainty, not reality. It is sharp when it separates possibilities that matter and when the reachable evidence can answer it.

Shape decides the answer space. A binary question decides. A comparison separates cases. A process question reveals sequence. A diagnostic question sorts cues into explanations. A causal question asks what changes the outcome. A counterfactual question asks what would have happened otherwise.

Size decides the grain. A broad question opens the territory. A small question enters the concrete instance. The wrong size either leaves the pattern blurry or loses it by zooming too far in.

Angle decides the standpoint. The same situation changes when seen from user experience, system boundary, or mechanism. It changes again under evidence strength or incentive. Time and failure can change it too. Make the angle visible so the user can see what the question reveals and what it hides.

Depth decides the layer. Surface description is not mechanism. Mechanism is not counterfactual dependence. Deeper means closer to the structure that explains or changes the target.

Always name the limit. A question can cut cleanly and still reveal only one face. Return the companion question or check that keeps the inquiry from mistaking that face for the whole.

## Avoid False Discovery

Watch for answer-smuggling. A question fails when it already contains the conclusion, hides the preferred frame, or makes alternatives hard to say.

Watch for attention leak. A question also fails when it replaces the user's target with a socially safer, morally cleaner, or more fashionable target. The repair is to state the side frame as out of scope unless it is needed to answer the accepted question.

Watch for false depth. A question using why-language can still stay at association. Mechanism appears only when the inquiry can say what change would produce a different result.

Watch for premature framing. When the problem definition is unstable, compare frames before executing inside the first one.

Watch for surface interviewing. Tacit knowledge rarely appears through abstract explanation. Move into concrete incidents where cues and choices become visible. Then ask what surprised the person. Ask which option they rejected and which condition changed the action.

Watch for boundary blindness. If the target behaves like a system, isolated parts will omit the relation that matters. Name the boundary and ask what changes when the boundary moves.

Watch for false contact. A fluent answer can still be only a public account, a memory repair, or the story allowed by the setting. Escalate the rung when the user's consequence requires closer contact with what happened.

## Use Other Skills When Needed

Use intention-capture when the latest message does not reveal why the inquiry matters. It should clarify the target and consequence. It should also clarify the audience or commitment state when those would change the inquiry path. Question Sharpener turns that captured reason into a discovery path.

Use expert-research when the inquiry depends on current facts or outside evidence. Use it when formal literature or domain expertise can sharpen the question. Use it when the consequence is high enough that a weak premise would damage the inquiry. Expert-research supplies the evidence boundary and domain vocabulary. Question Sharpener still decides what the inquiry is trying to uncover and which instrument should uncover it.

Use writer whenever the final inquiry question or probe will be shown to a person. Question Sharpener decides the discovery path and the reveal instrument. It also protects the check instrument and the limit. Writer shapes the wording so the question reaches the reality of the situation without sounding generic, performative, or borrowed.

Run the chain in this order. First, use intention-capture when the user's discovery purpose could be misread. Second, use Question Sharpener to set the target and instrument. Third, use expert-research if outside reality must sharpen the premise. Fourth, use writer for the final question wording. If writer softens away the reveal or changes the instrument, restore the inquiry logic and rewrite the line.

Use skillsmith only when the discovery method itself is being turned into a portable skill.

## Validate

Before returning, inspect five gates together.

The intention gate asks whether the inquiry serves the user's actual discovery purpose rather than the literal surface of the request. If the purpose is uncertain enough to change the path, use intention-capture or ask the narrow question it would ask.

The target gate asks whether the hidden thing is named clearly enough that inquiry can reduce it. The instrument gate asks whether the chosen instrument fits the kind of hiddenness. The coverage gate asks whether the inquiry sees enough perspectives for the user's consequence. The distortion gate asks whether leading wording or premature framing still controls the path. The return gate asks whether the output shows what is known and what is inferred. It also asks what remains uncertain and what is worth testing next.

The reality-contact gate asks whether the inquiry is close enough to the thing it claims to reveal. If the answer could be performative or remembered poorly, move toward a concrete incident. If the answer is socially risky, protect disclosure and look for safer evidence. If the answer is domain-naive, use expert-research before trusting the premise. If action can safely teach the answer, use a small test.

The return gate should reject outputs that overwhelm the user with candidate questions. It should also reject outputs that skip past discovery into the artifact the discovery was meant to inform.

The attention-leak gate asks whether every visible move helps answer the accepted question, reveal the hidden target, or protect the chosen contact rung. If a move mainly polices the user's premise, remove it. If a hard boundary is genuinely required, state the boundary as a constraint on the inquiry rather than as a judgment of the user.

For ordinary requests, more than five visible questions is usually a sign that the output has become an interview script. Compress it back into the discovery path.

The wording gate asks whether the final human-facing question has passed through writer. The question should still cut the right uncertainty. It should make reality easier to answer from lived evidence, not sound like a generic survey prompt.

The wording gate should also reject overloaded questions. If one question contains many possible answers, repair it into a simpler reality-contact question and keep the possibilities as internal listening notes.

Do the same for probes. A probe should ask for the next piece of reality, not offer a menu of explanations.

In the returned surface, do not put answer examples after the question unless the user asked for options. Put those possibilities under a listening lens, or keep them internal.

When the inquiry path claims readiness, use an independent cold reader. Give a fresh subagent only the final question or discovery path plus the intended setting when that setting is needed. Do not give the grounding or rejected-frame notes. Keep the suspected fix out too. The reader should be able to say what the inquiry tries to reveal. They should also recover the hiddenness type. They should see the instrument and check. They should name the contact rung and main distortion risk from the returned surface alone. A same-session cold read can guide repair, but it is diagnostic only.

Passing these gates does not prove the discovery is true. It means the inquiry path is fit enough to begin or to hand off.

## Return Shape

Return the smallest surface that helps the user uncover the thing.

For a sharper-question request, give the revised question. Then name its angle and its reveal. Name what it hides and the companion check.

For a discovery-plan request, give the target and hiddenness type. Then give the primary instrument and check instrument. Add the inquiry sequence, coverage lens, and next test.

When the contact rung matters, name it in the return. Say whether the path relies on a stated account or a concrete incident. When the path needs stronger contact, name the source of that contact explicitly so the user can see why the inquiry moved closer to reality.

For an existing inquiry, name the blind spot and repair the next move.

For larger discovery work, include a compact discovery map:

```text
Target:
Current cover:
Primary instrument:
Check instrument:
Inquiry sequence:
Perspectives covered:
Likely distortion:
What a good answer should show:
Next decisive test:
```

Keep the language plain. The user should feel the covered thing becoming easier to uncover.
